# Requisitos No Funcionales

## RNF-01 Usabilidad
La aplicación debe ser fácil de utilizar para productores, compradores, visitantes y miembros del comité.

## RNF-02 Diseño responsive
El sistema debe adaptarse correctamente a computadoras, tablets y dispositivos móviles.

## RNF-03 Rendimiento
Las pantallas principales deben cargar de forma rápida para mejorar la experiencia del usuario.

## RNF-04 Seguridad
La información de los usuarios debe manejarse de forma segura dentro del sistema.

## RNF-05 Compatibilidad
La aplicación debe funcionar correctamente en navegadores modernos como Chrome, Edge y Firefox.

## RNF-06 Mantenibilidad
El código debe estar organizado para facilitar futuras mejoras y correcciones.

## RNF-07 Escalabilidad
La estructura del sistema debe permitir agregar nuevas funcionalidades en futuras versiones.

## RNF-08 Claridad de mensajes
Los mensajes mostrados por el sistema deben ser claros y fáciles de entender para el usuario.

## Observación
Estos requisitos ayudan a que el sistema sea más fácil de usar, mantener y mejorar en futuras versiones.
